# 工作流编辑与运行体验增强-实现记录

## 任务需求
参考 Make 工作流编辑器的交互，补齐工作流页的保存反馈、自动排列、流程解释、请求拖拽编排、节点搜索定位和运行日志能力，提升编排与排障效率。

## 实现方案
1. 重构工作流页面布局。
   - 保留左侧工作流列表。
   - 在工作流列表与主画布之间新增辅助入口区，提供“请求 / 搜索 / 日志 / 解释”切换面板。
   - 在画布底部新增悬浮操作条，承载保存状态、重试保存、自动排列、重置视角和流程解释开关。

2. 强化保存链路。
   - 继续复用现有工作流自动保存接口。
   - 将保存状态细化为未保存、保存中、已保存、保存失败。
   - 增加手动重试保存逻辑，并在运行工作流前强制等待待保存内容落盘。

3. 增加工作流分析与布局工具。
   - 新增 `src/utils/workflowEditor.ts`，统一处理执行链路分析、自动排列、流程解释、日志排序和敏感请求头脱敏。
   - 自动排列按触发器主链路分层布局，孤立节点单独落位，异常结构走回退顺序。

4. 补齐请求拖拽、搜索和解释能力。
   - 请求面板支持拖拽到画布创建节点，也保留点击添加。
   - 搜索面板支持按当前工作流节点名称实时过滤并定位。
   - 解释面板基于当前工作流配置生成摘要和步骤说明，并与画布节点联动。

5. 增加运行日志持久化。
   - 服务端新增工作流运行日志存储与读取接口。
   - 日志按工作流保存运行摘要和节点级快照。
   - 对鉴权头、Cookie 等敏感信息做脱敏后再持久化和展示。
   - 前端新增日志列表与日志详情抽屉，支持查看节点级执行明细。

## 开发过程精要
1. 扩展工作流类型定义，补充解释、运行日志和执行快照结构。
2. 修正 `workflowStore` 删除节点时未同步删除关联边的问题。
3. 重写工作流页核心交互，整合保存、运行、拖拽、搜索、解释和日志。
4. 调整 `WorkflowSidebar`、`WorkflowToolbar`、`WorkflowResultsPanel`、`WorkflowNodeDetail` 的接口与展示逻辑。
5. 为服务端日志规范化增加 `node:test` 覆盖，并完成类型检查。
