# bug 文档

# 说明
bug 文档中保存实际使用中遇到的 bug 问题和该 bug 关联的需求（如果是从需求中产生的话），agent 开始处理 bug 前需要对 bug 产生的原因进行分析，分析完后整理解决方案，再开始修复 bug，修复 bug 的过程中需要查看之前已经遇到和修复过的 bug，避免产生同样的 bug。对于已经修复的 bug，需要单独在 /bug 文件夹下添加一个文件来记录 bug 的出现原因和解决方案。
如果 bug 是从需求中产生的，会在需求关联部分提供需求的序号，需要 agent 根据需求序号到 plan.md 中查找对应需求，再在 /plan 文件夹下找到对应需求的文档，结合文档中的开发方案一起修复 bug。

例如：
bug：
[] 1 连线样式错误
bug 描述
工作流中各个节点之间的连线样式仍然是实线，需要修复为使用虚线连接。
需求关联 1

agent 修复完成后，首先将 bug.md 文件中这个 bug 任务前面的勾打上，再在 /bug 文件夹下创建文件 连线样式错误.md，内容则写 bug 出现原因，解决方案和开发过程精要。

# 注意
此文档只可以修改任务的完成状态，即修改任务前面的复选框[]
